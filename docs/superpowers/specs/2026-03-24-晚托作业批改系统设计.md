## 一、项目概述

### 1.1 项目背景

服务于小学生晚托辅导机构，提供作业批改、错题管理、学情分析等功能的双向平台（老师端 + 学生/家长端）。

### 1.2 目标用户

- **小型机构**: 1-2 个校区，老师≤10 人，学生≤100 人
- **用户角色**: 管理员、主讲老师、辅导老师、家长、学生

### 1.3 核心价值

1. 提高作业批改效率
2. 自动归集错题，减少人工整理
3. 基于艾宾浩斯记忆曲线的科学复习
4. 一键生成 A4 错题集，方便巩固

### 2.1 技术架构

```
┌─────────────────────────────────────────────────────────────┐
│                    响应式网页 (Web App)                      │
│  ┌─────────────────────┐    ┌─────────────────────────────┐ │
│  │     老师端           │    │       家长/学生端            │ │
│  └─────────────────────┘    └─────────────────────────────┘ │
└─────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────┐
│                    后端服务 (Next.js API)                    │
└─────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────┐
│                    数据库 (PostgreSQL)                       │
└─────────────────────────────────────────────────────────────┘
```

### 2.2 技术栈

| 层级 | 技术 | 理由 |
|------|------|------|
| 前端框架 | Next.js 14 (App Router) | 前后端一体、部署简单 |
| UI 组件 | TailwindCSS + shadcn/ui | 快速搭建、响应式 |
| 数据库 | PostgreSQL (Supabase) | 免费额度够用、自带管理后台 |
| 部署 | Vercel | 一键部署、免费 HTTPS |
| 图片存储 | Supabase Storage | 便宜、易集成 |
| 通知 | 企业微信机器人 Webhook | 免费、简单 |

## 五、数据库设计

### 5.1 核心表结构

```sql
-- 用户表
CREATE TABLE users (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    name VARCHAR(100) NOT NULL,
    phone VARCHAR(20) UNIQUE,
    role VARCHAR(20) NOT NULL CHECK (role IN ('admin', 'head_teacher', 'teacher', 'parent', 'student')),
    enterprise_wechat_id VARCHAR(100),
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 班级表
CREATE TABLE classes (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    name VARCHAR(100) NOT NULL,
    description TEXT,
    status VARCHAR(20) DEFAULT 'active' CHECK (status IN ('active', 'archived')),
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 班级 - 学生关联表（支持调班）
CREATE TABLE class_students (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    class_id UUID REFERENCES classes(id),
    student_id UUID REFERENCES users(id),
    joined_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    left_at TIMESTAMP,
    status VARCHAR(20) DEFAULT 'active' CHECK (status IN ('active', 'transferred')),
    UNIQUE(class_id, student_id, status)
);

-- 老师 - 班级关联表
CREATE TABLE class_teachers (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    class_id UUID REFERENCES classes(id),
    teacher_id UUID REFERENCES users(id),
    role VARCHAR(20) DEFAULT 'head_teacher' CHECK (role IN ('head_teacher', 'assistant')),
    UNIQUE(class_id, teacher_id)
);

-- 作业表
CREATE TABLE assignments (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    student_id UUID REFERENCES users(id),
    subject VARCHAR(50) NOT NULL CHECK (subject IN ('math', 'chinese', 'english')),
    image_urls TEXT[] NOT NULL,
    description TEXT,
    status VARCHAR(20) DEFAULT 'pending' CHECK (status IN ('pending', 'completed')),
    uploaded_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    completed_at TIMESTAMP
);

-- 批改记录表
CREATE TABLE corrections (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    assignment_id UUID REFERENCES assignments(id),
    teacher_id UUID REFERENCES users(id),
    score DECIMAL(5,2),
    grade VARCHAR(10),
    comment TEXT,
    is_wrong_question BOOLEAN DEFAULT FALSE,
    knowledge_point_id UUID,
    corrected_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    notified BOOLEAN DEFAULT FALSE
);

-- 知识点体系表
CREATE TABLE knowledge_points (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    subject VARCHAR(50) NOT NULL CHECK (subject IN ('math', 'chinese', 'english')),
    grade VARCHAR(50) NOT NULL,
    unit VARCHAR(100),
    name VARCHAR(200) NOT NULL,
    parent_id UUID REFERENCES knowledge_points(id),
    path TEXT,
    UNIQUE(subject, grade, unit, name)
);

-- 错题表
CREATE TABLE wrong_questions (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    student_id UUID REFERENCES users(id),
    correction_id UUID REFERENCES corrections(id),
    knowledge_point_id UUID REFERENCES knowledge_points(id),
    image_url VARCHAR(500) NOT NULL,
    correct_answer TEXT,
    review_count INTEGER DEFAULT 0,
    last_review_at TIMESTAMP,
    mastery_status VARCHAR(20) DEFAULT 'new' CHECK (mastery_status IN ('new', 'reviewing', 'mastered')),
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 复习计划表
CREATE TABLE review_plans (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    student_id UUID REFERENCES users(id),
    generated_date DATE NOT NULL,
    wrong_question_ids UUID[] NOT NULL,
    status VARCHAR(20) DEFAULT 'pending' CHECK (status IN ('pending', 'completed')),
    teacher_confirmed_at TIMESTAMP,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

## 六、核心业务流程

### 6.1 作业批改流程

```
1. 学生/家长上传作业图片
   ↓
2. 老师收到待批改通知
   ↓
3. 老师查看作业并批改（打分 + 评语）
   ↓
4. 老师可选择"标记为错题"
   ↓
5. 系统自动归集到学生错题本
   ↓
6. 老师手动选择知识点标签
   ↓
7. 批改完成，通知家长
```

### 6.2 错题本生成流程

```
1. 老师批改时勾选"标记为错题"
   ↓
2. 系统自动创建错题记录
   ↓
3. 老师从知识点树中选择对应标签
   ↓
4. 错题归集到学生错题本
   ↓
5. 系统根据艾宾浩斯曲线计算复习时间
```

### 6.3 A4 错题集生成流程

```
1. 老师选择学生、日期范围、科目
   ↓
2. 系统筛选符合条件的错题
   ↓
3. 生成 A4 格式 PDF（每题包含：原题、知识点、正确答案）
   ↓
4. 老师下载打印
```

## 七、界面设计要点

### 7.1 老师端核心页面

1. **仪表盘** - 待批改作业数、今日完成、班级概况
2. **作业批改页** - 图片展示、打分输入、评语框、错题标记按钮、知识点选择器
3. **班级管理页** - 学生列表、调班操作
4. **错题集生成页** - 筛选条件、预览、PDF 下载

### 7.2 家长/学生端核心页面

1. **作业列表** - 按日期展示、状态筛选
2. **作业详情** - 图片、批改结果、老师评语
3. **错题本** - 按科目/知识点筛选、复习状态
4. **复习计划** - 每日推荐复习题目

## 八、安全与隐私

1. 用户认证：JWT Token，7 天有效期
2. 数据隔离：老师只能查看负责班级的数据
3. 图片存储：私有 bucket，签名 URL 访问
4. 敏感信息：手机号加密存储
